# 反家暴援助小程序 — 项目开发文档

> 版本：v1.0 | 日期：2026-07-11

---

## 一、项目概述

### 背景与目标
为家庭暴力受害者提供一个**安全、匿名、即时可用**的援助工具。核心价值：
- 让受害者在隐蔽状态下获得帮助（伪装保护）
- AI 陪伴引导，降低求助门槛
- 提供法律知识、证据收集、资源匹配的一站式支持

### 核心设计原则
1. **隐蔽优先**：任何功能都不能暴露用户正在使用反家暴工具
2. **匿名第一**：不绑定真实身份，不留可追踪痕迹
3. **数据安全**：证据加密存储，本地不留明文副本
4. **低门槛**：面向低技术、高压力状态下的用户，操作极简

## 二、功能需求

### 0. 伪装页（Disguise Layer）

**功能描述：**
小程序启动时默认显示无害外观（如"简易记账本"），保护用户不被发现。

**详细需求：**
- 伪装页外观：记账本 UI，支持正常输入几笔记录（降低可疑度）
- 解锁方式：在"金额"输入框输入预设密码（默认为用户自定义数字），回车后跳转真实首页
- 密码设置：在「我的 → 隐私设置」中修改，支持 4-8 位数字
- 连续错误不提示、不锁定（避免暴露存在隐藏功能）
- 支持自定义触发方式（进阶）：如连点右上角 3 次

**注意事项：**
- 禁止在小程序名称/图标上出现任何家暴相关词汇
- 建议小程序名称：「日记本」「备忘小册」「我的账单」等

## 三、AI 智能体模块设计

### 模型选型（基于 openai-next.com 平台）

| 模块 | 使用模型 | 理由 |
|------|----------|------|
| 对话主模型 | `claude-sonnet-4-6` | 情感理解强，同理心最佳，内容安全可靠 |
| 多模态/图片分析 | `qwen3-vl-max` | 中文场景优化，伤情图片识别准确，成本低 |
| 法律 RAG 生成 | `deepseek-v3` | 中文法律文本理解强，适合知识库检索生成 |
| 风险评估推理 | `deepseek-r1` | 结构化推理，按评估量表逐步判断 |
| 证据整理生成 | `claude-sonnet-4-6` | 长文输出质量高，格式规范 |

**统一 API 接口：**
```
Base URL: https://api.openai-next.com/v1
格式：兼容 OpenAI Chat Completions API
```

### Agent 模块详细设计

#### 模块 1：情感对话 Agent
```
职责：主对话，情感支持，引导用户描述情况
输入：用户文字/语音转文字
输出：共情回应 + 下一步建议
System Prompt 要点：
  - 你是一位专业的家暴危机干预支持者
  - 始终相信用户的描述
  - 不建议"沟通解决"，不做道德评判
  - 检测到危险词汇时主动推送求助资源
触发危机词汇示例：杀我、活不下去、拿刀、逃不掉
```

#### 模块 2：法律知识 RAG Agent
```
职责：回答法律相关问题
知识库内容：
  - 《反家庭暴力法》全文
  - 人身安全保护令申请流程
  - 离婚诉讼中家暴的证据标准
  - 各地法援机构联系方式
技术栈：
  - Embedding 模型：text-embedding-3-small
  - 向量数据库：Qdrant（自部署）
  - 检索：Top-K 相似度匹配 + Rerank
  - 生成：deepseek-v3
```

#### 模块 3：风险评估 Agent
```
职责：评估当前危险等级，给出具体建议
流程：
  1. 呈现结构化问卷（参考 DA Risk Assessment）
  2. 用户选择/填写
  3. deepseek-r1 推理得出危险等级（低/中/高/极高）
  4. 对应推出建议行动清单
问卷维度：
  - 暴力频率与严重程度
  - 是否使用过凶器
  - 是否威胁过生命
  - 孩子是否在场
  - 是否有离开的条件
```

#### 模块 4：证据整理 Agent
```
职责：帮用户整理证据，生成可用于法律程序的文档
输入：用户上传的图片、录音描述、文字陈述
输出：
  - 时间线整理（按发生顺序排列）
  - 伤情描述（专业描述语言）
  - 证据清单（编号、类型、日期）
  - 书面陈述草稿（可直接用于报警/起诉）
模型：claude-sonnet-4-6（长文本输出质量高）
```

#### 模块 5：多模态图片分析 Agent
```
职责：分析伤情图片，提供专业描述
输入：用户上传的伤情照片
输出：
  - 伤情描述（部位、严重程度、推测伤害方式）
  - 建议就医提示
  - 是否建议立即报警
模型：qwen3-vl-max
注意：输出结果作为辅助参考，非医疗诊断
```

## 五、数据安全设计

### 证据库加密方案
```
1. 用户上传图片/文件
2. 后端生成随机 AES-256 密钥（每个用户独立）
3. 文件加密后上传 COS
4. 密钥用用户匿名 ID 派生的密钥二次加密存储
5. 用户本地不存任何明文
```

### 隐私保护清单
- [ ] 图片上传前自动去除 EXIF（GPS、设备信息）
- [ ] 对话内容不用于模型训练（在 API 调用时设置参数）
- [ ] 不收集手机号、真实姓名
- [ ] 日志不记录对话内容，只记录请求时间和模型
- [ ] 支持一键销毁账号和全部数据

## 八、关键风险与应对

| 风险 | 应对方案 |
|------|----------|
| 小程序审核被拒（涉及敏感话题）| 申请时使用公益机构名义，与妇联/NGO合作背书 |
| 施暴者伪装受害者进入社群 | 行为分析 + 人工审核 + 一键举报 |
| AI 给出不当建议 | 设定严格 System Prompt + 输出过滤 + 免责声明 |
| 数据泄露 | 端到端加密 + 不存明文 + 定期安全审计 |
| 服务器被攻击导致数据泄露 | 证据加密存储，即使服务器被攻击也无法读取明文 |
| 用户在使用时被发现 | 伪装页 + 快速离开按钮 + 截图防护 |

## 九、接口设计（核心）

### AI 对话接口
```
POST /api/chat
{
  "session_id": "匿名会话ID",
  "message": "用户输入",
  "type": "text|image|voice",
  "image_url": "可选，图片地址"
}

Response:
{
  "reply": "AI回复",
  "is_crisis": false,
  "crisis_level": 0-3,
  "action_card": null | { "type": "seek_help|save_evidence", "content": {} }
}
```

### 风险评估接口
```
POST /api/risk-assessment
{
  "answers": { "q1": 2, "q2": 1, ... }
}

Response:
{
  "risk_level": "low|medium|high|critical",
  "score": 75,
  "suggestions": ["立即离开", "拨打12338", ...],
  "resources": [{ "name": "广州家暴庇护所", "phone": "xxx" }]
}
```

### 证据上传接口
```
POST /api/evidence/upload
{
  "session_id": "匿名会话ID",
  "file": "文件",
  "description": "用户描述",
  "incident_time": "2026-07-10"
}

Response:
{
  "evidence_id": "加密ID",
  "ai_description": "AI生成的专业伤情描述",
  "stored": true
}
```
